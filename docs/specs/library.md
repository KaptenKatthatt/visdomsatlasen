# Library

## Purpose

The Library is the part of Visdomsatlasen intended for deliberate exploration.

It contains the wider collection of:

- reflection rooms
- original texts
- sources
- traditions
- historical context
- people
- questions
- themes

The Library supports curiosity without turning Visdomsatlasen into an endless content feed.

It is secondary to the Reading Room.

A user should never need to enter the Library to have a complete and meaningful experience.

---

## Core Principle

> The Reading Room helps the user stay with one thought. The Library helps the user intentionally find another.

The Library must never pull the user away from the Reading Room.

The user enters the Library through a conscious choice.

---

## Relationship to the Reading Room

The Reading Room and the Library serve different purposes.

### Reading Room

- one thought
- one room
- few decisions
- minimal navigation
- reflection
- stillness

### Library

- deliberate exploration
- access to sources
- broader context
- search
- comparison
- choice

The visual and functional distinction between them should be clear.

The Library may contain more information, but it must still feel calm and restrained.

---

## Entry

The Library should be accessible through a persistent but visually quiet navigation option.

Suggested Swedish label:

> Biblioteket

The Library must not be presented as the primary action on the application home screen.

Entering it should feel like standing up from the reading chair and walking toward the shelves.

---

## Library Home

The Library home should provide a small number of clear entry points.

Sections, in order (editorial decision 2026-07-18):

- Traditions
- Sources
- Themes
- Paths
- Rooms
- Questions

The interface should use Swedish labels:

- Traditioner
- Källor
- Teman
- Vandringar
- Rum
- Frågor

Not every section needs to be equally prominent.

Traditions and sources open the page as the library's calm frame — the shelves
one walks past first. Questions are gathered behind a single entry point
("Alla frågor") at the bottom, with the full list on its own subpage, so the
landing page stays short.

Saved items are not a Library home section — they are reached through the
Sparat tab in the main navigation.

---

## Primary Organization

The Library home presents content in this order (editorial decision 2026-07-18,
replacing the earlier "human experience first" ordering):

1. Traditions
2. Original sources
3. Themes
4. Paths
5. Reflection rooms
6. Human questions (as a single "Alla frågor" entry point)

Traditions and sources form the calm, stable frame of the library; questions
remain an important entry point, but the full list lives on its own subpage
rather than dominating the landing page. Elsewhere in the app — the reading
room, search, theme pages — questions and themes keep their central role:
Visdomsatlasen is still organized around human experience rather than
authority or authorship.

---

## Questions

Questions are a central way into the Library's content. On the Library home
they are gathered behind a single "Alla frågor" entry; the full list lives on
its own subpage.

Examples:

- What gives life meaning?
- How do we live with uncertainty?
- What is courage?
- Why do people seek power?
- How should we face death?
- What does forgiveness require?

In the Swedish interface, questions should be written naturally:

- Vad ger livet mening?
- Hur lever man med osäkerhet?
- Vad är mod?
- Varför söker människor makt?
- Hur möter man döden?
- Vad kräver förlåtelse?

A question page may contain:

- a short description
- available reflection rooms
- related themes
- source material
- optional historical context

It must not become an automatic reading sequence.

---

## Themes

Themes provide broad and emotionally understandable groupings.

Examples:

- Calm
- Meaning
- Courage
- Truth
- Suffering
- Humanity
- Forgiveness
- Responsibility
- Freedom
- Mortality

Themes are broader than questions.

A theme may contain several questions and rooms.

The number of visible themes should remain limited.

Avoid creating a long taxonomy that feels administrative or academic.

---

## Reflection Rooms

The Library may display all published reflection rooms.

Each room preview should contain only:

- title
- short summary
- theme or question
- approximate reading time

Do not display:

- popularity
- number of readers
- completion status
- rankings
- engagement statistics
- trending labels

Room previews should be visually quiet.

They should not compete for attention.

---

## Sources

The source section provides access to the original material behind the reflection rooms.

Sources may include:

- books
- scripture
- letters
- speeches
- philosophical works
- historical documents
- poetry
- oral traditions
- verified fragments

Each source record should contain:

- title
- author or attributed origin
- tradition
- approximate date
- edition or translation
- source location
- copyright or usage status
- connected reflection rooms

Sources should be presented accurately and without unnecessary praise or authority language.

---

## Traditions

Traditions exist as a secondary way to explore the Library.

Examples:

- Christianity
- Stoicism
- Buddhism
- Taoism
- Judaism
- Ancient Greek philosophy
- Egyptian religion
- Humanism

Traditions must not dominate the Library structure.

They help users understand context.

They do not own the questions or ideas.

In the Swedish interface, tradition pages may use labels such as:

- Kristendom
- Stoicism
- Buddhism
- Taoism
- Judendom
- Antik grekisk filosofi
- Egyptisk religion
- Humanism

---

## People and Authors

People and authors are reference points.

They are not the primary navigation.

A person page may contain:

- short biography
- historical period
- important works
- connected questions
- connected rooms
- source references

Avoid prominent name-dropping in the main experience.

Names become useful when the user chooses to understand origin and context.

---

## Search

Search should support intentional discovery.

It should search across:

- questions
- themes
- room titles
- room summaries
- source titles
- source references
- traditions
- people
- tags

Search results should prioritize:

1. Questions
2. Themes
3. Rooms
4. Sources
5. Traditions
6. People

The search experience must not behave like a content feed.

Results should be finite, clearly grouped and easy to scan.

---

## Search Interface

Search should be simple.

A single field is preferred.

Suggested Swedish placeholder:

> Sök efter en fråga, tanke eller källa

Advanced filters should not be visible by default.

Optional filters may be revealed when needed:

- Theme
- Source type
- Tradition
- Historical period

Avoid forcing users to configure filters before seeing useful results.

---

## Browsing

Browsing should be finite.

Do not use infinite scrolling.

Use one of the following:

- short sections
- pagination
- explicit “Visa fler”
- alphabetical or thematic indexes

The user should always understand how much content remains.

The Library must not create the sensation of an endless feed.

---

## Content Density

The Library may be more information-rich than the Reading Room, but it should not feel crowded.

Each screen should have:

- a clear purpose
- a clear hierarchy
- restrained metadata
- generous spacing
- limited simultaneous choices

When a page contains too many options, divide it into clearer sections rather than reducing spacing.

---

## Recommendations

The Library must not use algorithmic recommendations designed to increase engagement.

Allowed forms of connection include:

- editorially related questions
- rooms connected to the same theme
- sources used in the current room
- historically relevant context

These should be described neutrally.

Preferred Swedish wording:

- Närliggande frågor
- Källor i detta rum
- Fler perspektiv på frågan
- Historisk bakgrund

Avoid:

- Du kanske också gillar
- Populärt just nu
- Rekommenderat för dig
- Andra läser också

---

## Opening a Room from the Library

When the user selects a reflection room, the application should enter Reading Room mode.

The Library interface should disappear.

The user should not remain surrounded by search results, filters or related content.

The reading experience must remain protected.

When the room ends, the user may choose to return to the previous Library location.

---

## Saved Items

The Library may contain a quiet saved section.

Suggested Swedish label:

> Sparat

Saved items may include:

- reflection rooms
- sources
- questions
- personal notes

Saving is equivalent to placing a bookmark in a book.

It must not become a collection mechanic.

Do not display totals as achievements.

Avoid language such as:

- collection
- completed
- unlocked
- progress

---

## Empty States

Empty states should be calm and direct.

Example for saved items:

> Du har inte sparat något ännu.

No pressure should be added.

Do not use language such as:

> Börja bygga din samling idag.

---

## Mobile Experience

The Library is mobile first.

On smaller screens:

- show one clear section at a time
- avoid dense multi-column layouts
- keep filters collapsed by default
- use comfortable touch targets
- preserve readable spacing
- avoid fixed UI that reduces reading space

The Library should remain useful without feeling like a compressed desktop database.

---

## Desktop Experience

On larger screens, the Library may use additional horizontal space.

Possible layouts include:

- restrained two-column layouts
- a narrow navigation index beside content
- wider search result lists
- contextual source panels

Desktop layouts must not increase the number of competing focal points.

---

## Accessibility

The Library must support:

- semantic headings
- keyboard navigation
- visible focus states
- screen readers
- clear labels
- sufficient contrast
- scalable text
- reduced motion

Search results and filters must be understandable without relying on color or visual position alone.

---

## Acceptance Criteria

The Library is correctly implemented when:

- [ ] It is clearly secondary to the Reading Room.
- [ ] Questions and themes are more prominent than authors or traditions.
- [ ] Search results are finite and grouped.
- [ ] Infinite scrolling is not used.
- [ ] No popularity or engagement metrics are displayed.
- [ ] Opening a room removes Library distractions.
- [ ] Sources remain easy to locate and verify.
- [ ] Users can intentionally browse without being pushed forward.
- [ ] The experience remains calm on mobile.
- [ ] Saved items feel like bookmarks rather than achievements.

---

## Out of Scope

The Library must not include:

- social feeds
- user comments
- likes
- follower systems
- trending content
- popularity rankings
- algorithmic engagement recommendations
- autoplay
- reading streaks
- completion percentages
- content unlocks
- endless scrolling

These features conflict with the purpose of Visdomsatlasen.
