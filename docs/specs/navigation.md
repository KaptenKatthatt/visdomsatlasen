# Navigation

## Purpose

Navigation in Visdomsatlasen should feel effortless.

The application should never resemble a website with many pages or a knowledge base with complex menus.

Instead, navigation should feel like moving quietly through a physical place.

Every movement should have intention.

Every destination should feel meaningful.

---

# Core Principles

Navigation exists to support reflection.

Never exploration for its own sake.

Never engagement.

Never endless browsing.

The goal is to reduce decisions, not increase them.

---

# The Reading Room

The Reading Room is the heart of the application.

It is where every visit begins.

The Reading Room contains one room.

One question.

One opportunity to slow down.

Nothing competes with it.

---

# The Library

The Library exists for intentional exploration.

Users visit the Library because they choose to.

The Reading Room never pushes users into the Library.

The Library never pulls users away from reflection.

---

# Navigation Hierarchy

Navigation follows this order:

Reading Room

↓

Question

↓

Room

↓

Reflection

↓

Sources

↓

Library

↓

Related Questions

Every step moves naturally from reflection toward deeper exploration.

Never the opposite.

---

# The Home Experience

Opening the application should feel immediate.

The user should not face a dashboard.

The user should not face a menu.

Instead, the user is gently invited into a single reading experience.

The application begins with presence, not navigation.

---

# Primary Navigation

Primary navigation should remain minimal.

Recommended destinations:

- Reading Room
- Library
- Notes
- Settings

Nothing else belongs in the main navigation unless it clearly improves the experience.

---

# Secondary Navigation

Secondary navigation should stay hidden until needed.

Examples include:

- Source details
- Historical timelines
- Related questions
- Glossary
- Search filters

Secondary navigation should never distract from reading.

---

# Search

Search is a tool.

Not a destination.

Users should never feel expected to search.

Search should quietly support curiosity.

---

# Related Questions

Related questions exist to deepen understanding.

They do not exist to increase session length.

Only a small number of carefully selected connections should be shown.

Quality is more important than quantity.

---

# Ending a Room

Every room should have a natural ending.

The application should never immediately suggest more content.

Instead, users may choose between:

- Returning to the Reading Room
- Opening the Library
- Exploring a related question
- Closing the application

Closing the application is always considered a successful outcome.

---

# Notes

Notes belong to the reader.

They should always be easily accessible.

They should never interrupt reading.

Writing should feel as natural as placing a bookmark inside a book.

---

# Bookmarks

Bookmarks are quiet.

They acknowledge something worth returning to.

They are not achievements.

They are not collections.

They are simply places the reader wishes to revisit.

---

# Navigation Should Never...

Navigation should never:

- create urgency
- create fear of missing out
- encourage binge reading
- encourage endless exploration
- reward frequent interaction
- compete with the content

---

# The Decision Test

Whenever a new navigation element is proposed, ask:

Does this reduce cognitive load?

Does this help the reader?

Could this destination instead remain hidden until needed?

Would removing it improve the experience?

If removing it improves the experience, remove it.

---

# Long-Term Vision

A user should eventually know Visdomsatlasen so well that navigation almost disappears.

The application should become familiar enough that the user no longer thinks about where to go.

Only about what they are reading.

Navigation has succeeded when it becomes almost invisible.
